## Codebase Patterns
<!-- Consolidate reusable patterns here -->
- **Module structure**: each core module has `index.ts` barrel, tests co-located as `*.test.ts`
- **Stub convention**: empty modules export `export {};` with JSDoc module comment
- **Type exports**: use `export type` (verbatimModuleSyntax), `.js` extensions in imports
- **CLISchema.commands[].schema**: `CommandSchema` has `.name`, `.flags` (Record<string, FlagSchema>), `.args` (CommandArgEntry[])
- **FlagSchema**: `.kind`, `.aliases`, `.description`, `.enumValues`
- **Error pattern**: throw `CLIError` for user-facing errors (not raw Error)
- **Test conventions**: em-dash in describes, `// ===` major separators, `// ---` minor, zero lifecycle hooks
- **GlobalThis injection**: use optional `globals` param (typed interface) instead of `vi.stubGlobal` for testing globalThis-dependent code — avoids cleanup/mutation issues
- **Runtime detection**: `GlobalForDetect` interface, check Bun before Node (Bun mimics Node's `process`)
