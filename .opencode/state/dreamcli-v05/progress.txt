## Codebase Patterns
<!-- Consolidate reusable patterns here -->
- **Module structure**: each core module has `index.ts` barrel, tests co-located as `*.test.ts`
- **Stub convention**: empty modules export `export {};` with JSDoc module comment
- **Type exports**: use `export type` (verbatimModuleSyntax), `.js` extensions in imports
- **CLISchema.commands[].schema**: `CommandSchema` has `.name`, `.flags` (Record<string, FlagSchema>), `.args` (CommandArgEntry[])
- **FlagSchema**: `.kind`, `.aliases`, `.description`, `.enumValues`
- **Error pattern**: throw `CLIError` for user-facing errors (not raw Error)
- **Test conventions**: em-dash in describes, `// ===` major separators, `// ---` minor, zero lifecycle hooks
- **GlobalThis injection**: use optional `globals` param (typed interface) instead of `vi.stubGlobal` for testing globalThis-dependent code â€” avoids cleanup/mutation issues
- **Runtime detection**: `GlobalForDetect` interface, check Bun before Node (Bun mimics Node's `process`)
- **Bash string literals**: biome flags `${...}` in single-quoted strings as `noTemplateCurlyInString` â€” suppress with `biome-ignore` when generating bash syntax
- **ErasedCommand test stubs**: `_execute` must return `RunResult` shape including `error: undefined`; use typed `[] as string[]` to avoid `never[]`
- **FlagKind union**: `'string' | 'number' | 'boolean' | 'enum' | 'array'` â€” no `'count'` variant
- **Auto-adapter factory**: `createAdapter(globals?)` in `auto.ts` â€” separate from `detect.ts` to keep detection deps-free; exhaustive switch on `Runtime` union for compile-time safety
- **Built-in subcommands**: use `command()`/`flag()` builders internally inside `CLIBuilder` methods, capture `this.schema` for schema access; schema snapshot is pre-registration (excludes the built-in command itself from generated output)
- **Exhaustiveness guard**: `default: { const _exhaustive: never = x; return _exhaustive; }` â€” zero-dep pattern for union switch exhaustiveness; compile-time error on unhandled variant, zero runtime overhead
- **Barrel export audit**: every type used in a public function's parameter or return position must be exported from `src/index.ts` â€” check module barrels vs main barrel for gaps
- **Code-gen in --json mode**: commands that produce raw text (scripts, generated code) must branch on `out.jsonMode` â€” use `out.json({ script })` for JSON mode, `out.log()` for normal; `out.log()` redirects to stderr in JSON mode
