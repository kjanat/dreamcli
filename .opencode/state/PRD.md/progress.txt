## Codebase Patterns

- **Formatter ownership**: dprint owns all formatting (TS/JS via biome plugin, JSON, JSONC, YAML, markdown, etc). Biome's JSON formatter is disabled to avoid conflicts. Biome handles linting only for JSON.
- **JSONC trailing commas**: biome override for JSONC parses trailing commas but does NOT enforce them in formatting (set to "none") to avoid dprint conflicts.
- **Build**: tsdown with `hash: false` for stable dts filenames. `exports: true` auto-patches package.json. Conditional exports include `types` entries for both import and require.
- **Module structure**: `src/core/<module>/index.ts` barrel pattern. `src/runtime/<adapter>.ts` for platform adapters.
- **Test pattern**: vitest with `src/**/*.test.ts` collocated tests. Dynamic imports in tests: `import('./module.js')` (with .js extension for NodeNext).
- **Error hierarchy**: `CLIError` base → `ParseError` (exit 2), `ValidationError` (exit 2). Discriminated by `name` property. `ErrorCode` union is extensible via `string & {}`. Type guards: `isCLIError`, `isParseError`, `isValidationError`.
- **Flag builder pattern**: `FlagBuilder<C extends FlagConfig>` — single phantom type param tracks `valueType` + `presence`. `FlagSchema` is the runtime descriptor. `declare readonly _config: C` for type branding (no `!` assertion). `flag.<kind>()` factory → immutable builder chain. `InferFlag<B>` / `InferFlags<R>` extract resolved types. `const` type param on `enum()` infers literals. Boolean flags implicitly default to `false`.
