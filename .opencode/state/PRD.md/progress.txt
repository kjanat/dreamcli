## Codebase Patterns

- **Context type parameter pattern**: `CommandBuilder<F, A, C>` — third phantom param `C extends Record<string, unknown> = Record<string, never>` tracks middleware context. `Record<string, never>` makes `ctx` property access yield `never` (type error) until middleware widens it. `ActionParams<F, A, C>` and `ActionHandler<F, A, C>` carry `C` through. All metadata methods (`.description()`, `.alias()`, etc.) preserve `C`; `.flag()` and `.arg()` preserve `C` but drop handler. Default `C` makes all pre-v0.4 code backward-compatible.

- **Formatter ownership**: dprint owns all formatting (TS/JS via biome plugin, JSON, JSONC, YAML, markdown, etc). Biome's JSON formatter is disabled to avoid conflicts. Biome handles linting only for JSON.
- **JSONC trailing commas**: biome override for JSONC parses trailing commas but does NOT enforce them in formatting (set to "none") to avoid dprint conflicts.
- **Build**: tsdown with `hash: false` for stable dts filenames. `exports: true` auto-patches package.json. Conditional exports include `types` entries for both import and require.
- **Module structure**: `src/core/<module>/index.ts` barrel pattern. `src/runtime/<adapter>.ts` for platform adapters.
- **Test pattern**: vitest with `src/**/*.test.ts` collocated tests. Dynamic imports in tests: `import('./module.js')` (with .js extension for NodeNext).
- **Error hierarchy**: `CLIError` base → `ParseError` (exit 2), `ValidationError` (exit 2). Discriminated by `name` property. `ErrorCode` union is extensible via `string & {}`. Type guards: `isCLIError`, `isParseError`, `isValidationError`.
- **Flag builder pattern**: `FlagBuilder<C extends FlagConfig>` — single phantom type param tracks `valueType` + `presence`. `FlagSchema` is the runtime descriptor. `declare readonly _config: C` for type branding (no `!` assertion). `flag.<kind>()` factory → immutable builder chain. `InferFlag<B>` / `InferFlags<R>` extract resolved types. `const` type param on `enum()` infers literals. Boolean flags implicitly default to `false`.
- **Arg builder pattern**: `ArgBuilder<C extends ArgConfig>` — phantom type tracks `valueType` + `presence` + `variadic`. Positional args default to `'required'` (unlike flags which default to `'optional'`). `arg.custom(parseFn)` infers return type of parse function. Variadic args wrap type in `T[]` via `ResolvedArgValue`. `ArgSchema` stores `parseFn` for custom parsing.
- **Command builder pattern**: `CommandBuilder<F, A>` — two phantom type params accumulate flag/arg builder records via intersection (`F & Record<N, B>`). `.flag()` / `.arg()` return new builders with expanded types; `.action(handler)` receives `ActionParams<F, A>` with `InferFlags<F>` and `InferArgs<A>`. Handler is dropped when new flags/args are added (type safety). `CommandSchema` is the runtime descriptor with flags map + ordered args list. `{}` default for initial accumulator is correct and intentional (biome-ignore). `Out` interface stub defined for handler typing; real impl in output-1.
- **Parser pattern**: Two-phase design: `tokenize(argv)` → `Token[]` (schema-agnostic), then `parse(schema, argv)` → `ParseResult` (schema-aware). Flag lookup built from `CommandSchema.flags` + aliases as `Map<string, [canonicalName, FlagSchema]>`. Value coercion per `FlagKind`/`ArgKind`. Array flags accumulate. Positionals mapped by `CommandSchema.args` ordering. All errors are `ParseError` with structured `code`/`details`/`suggest`.
- **Resolver pattern**: `resolve(schema, parsed)` → `ResolveResult` with `flags` + `args`. MVP chain: CLI → default. Flags resolved first, then args. Missing required values throw `ValidationError`. Multiple errors aggregated into single throw with `details.errors` array. Array flags default to `[]`. Optional flags get `undefined` with key present. Non-empty tuple type `[T, ...T[]]` used for error aggregation to avoid `!` assertions. `isNonEmpty()` type guard narrows `T[]` → `[T, ...T[]]`.
- **Help pattern**: `formatHelp(schema, options?)` → `string`. Reads `CommandSchema` to render sections: Usage, Description, Arguments, Flags, Examples. `HelpOptions` controls `width` (default 80) and `binName`. Flags sorted: short-aliased first, then alphabetical. Type hints per `FlagKind`; boolean suppresses hint and `(default: false)`. Word-wrapping via `wrapText()` at word boundaries with indent continuation. Flag descriptions include resolution source annotations: `[env: VAR]` and `[config: path]` when configured, ordered between description text and presence indicators (`[required]`/`(default: X)`).
- **Output pattern**: `createOutput(options?)` → `Out`. `WriteFn = (data: string) => void` is the sole I/O seam. `OutputChannel` implements `Out`; routes `log`/`info` → stdout writer, `warn`/`error` → stderr writer. `Verbosity: 'normal' | 'quiet'` — quiet suppresses `info`. `isTTY` boolean for downstream TTY-aware formatting. `noopWrite` fallback when no writers provided. `createCaptureOutput()` test helper returns `[Out, CapturedOutput]` with separate stdout/stderr arrays. `Out` interface stays in `command.ts` (contract); implementation in `core/output/`.
- **Testkit pattern**: `runCommand(cmd, argv, options?)` → `Promise<RunResult>`. Full pipeline: help detection → parse → resolve → handler → result. Never throws — all errors caught. `RunResult` has `exitCode`, `stdout[]`, `stderr[]`, `error`. `RunOptions` injects `env`, `verbosity`, `help` without process state. `invokeHandler` internal function is the sole phantom-type boundary cast. `HandlerParams` models the erased `ActionParams<F,A>`. Standalone function (not method) to avoid circular deps with `command.ts`.
- **CLI builder pattern**: `cli(name)` → `CLIBuilder` (immutable, no phantom types). `.version()`, `.description()`, `.command()` — all return new builders. Two execution paths: `.execute(argv, opts?)` → `Promise<RunResult>` (testable, no process state) and `.run(opts?)` → `Promise<never>` (production, uses `RuntimeAdapter`). Type-erased command storage via `ErasedCommand` closures — captures typed `CommandBuilder<F,A>` in closure, presents uniform `_execute()` to dispatcher. No `any` types needed. `formatRootHelp()` is separate from `formatHelp()` — different format (program overview vs command details). `buildCommandRunOptions()` handles `exactOptionalPropertyTypes` via conditional spreads.
- **Runtime adapter pattern**: `RuntimeAdapter` interface in `src/runtime/adapter.ts` — fields: `argv`, `env`, `cwd`, `stdout: WriteFn`, `stderr: WriteFn`, `isTTY`, `exit`. `createNodeAdapter(proc?)` wraps Node/Bun `process` global. `createTestAdapter(opts?)` provides injectable stub — exit throws `ExitError` by default. `CLIRunOptions.adapter?` injects into `.run()` — defaults to `createNodeAdapter()`. `NodeProcess` internal interface avoids `@types/node` dependency. I/O seam: adapter `stdout`/`stderr` are `WriteFn` (same type as output module), not raw streams.
- **Env resolution pattern**: `resolve(schema, parsed, options?)` accepts `ResolveOptions` with optional `env` record. Resolution chain: CLI → env → default. `coerceEnvValue()` returns discriminated `CoerceEnvResult` (`ok: true | false`) instead of throwing — allows error aggregation. Boolean env coercion is lenient: accepts `true/false/1/0/yes/no/""` (case-insensitive). Array env values are comma-separated. Element coercion recurses via `coerceEnvValue()` with element schema. Env errors use `TYPE_MISMATCH` / `INVALID_ENUM` validation codes. `CLIBuilder.run()` auto-sources `adapter.env` when no explicit `env` option provided.
- **Config resolution pattern**: `ResolveOptions.config` accepts a plain `Record<string, unknown>` (JSON object). Resolution chain: CLI → env → config → default. `resolveConfigPath(config, dottedPath)` walks nested objects segment-by-segment; returns `undefined` for missing/non-object intermediates. `coerceConfigValue()` returns discriminated `CoerceConfigResult` (same shape as env). Config values may already be typed — coercion is lenient for matching types (numbers stay numbers, booleans stay booleans) and stricter for mismatches. Array config values accept both actual arrays and comma-separated strings. File loading is the caller's responsibility.
- **Prompt schema pattern**: `PromptConfig` discriminated union (kind: confirm/input/select/multiselect) in `src/core/schema/prompt.ts`. `FlagSchema.prompt: PromptConfig | undefined` stores prompt config. `FlagBuilder.prompt(config)` metadata modifier (same immutable pattern as `.env()`/`.config()`). `PromptResult` discriminated on `answered: true/false` — coercion is resolver's responsibility. `SelectChoice` for select/multiselect options. `InputPromptConfig.validate` for inline validation (returns `true | string`). Prompt types re-exported through flag.ts → schema barrel → public surface.
- **Prompt engine pattern**: `PromptEngine` interface with `promptOne(config: ResolvedPromptConfig) → Promise<PromptResult>` — engine is a pure renderer, no schema knowledge. `ResolvedPromptConfig` variant guarantees non-empty choices for select/multiselect (merged from `FlagSchema.enumValues` by `resolvePromptConfig()`). `ReadFn = () => Promise<string | null>` is the stdin seam (null = EOF). `createTerminalPrompter(read, write)` for line-based I/O. `createTestPrompter(answers, options?)` for testing with queue-based answers. `PROMPT_CANCEL` symbol sentinel for cancelled prompts in test queue. `MAX_RETRIES = 10` safety valve on all terminal prompts.
- **Prompt resolution pattern**: `resolve()` is now async (`Promise<ResolveResult>`). Resolution chain: CLI → env → config → **prompt** → default. `ResolveOptions.prompter?: PromptEngine` — when present and flag has `schema.prompt`, calls `resolvePromptConfig()` → `prompter.promptOne()` → `coercePromptValue()`. Three-outcome `PromptResolveResult`: `ok: true` (value), `ok: false, error` (coercion fail), `ok: false, no error` (cancelled → falls through to default/required). Non-interactive mode: no prompter → prompt step skipped entirely. `coercePromptValue()` mirrors env/config coercion but prompt-specific error messages with `source: 'prompt'` in details. `RunOptions.prompter` and `CLIRunOptions.prompter` thread prompter through testkit and CLI builder into resolve.
- **Runtime stdin pattern**: `RuntimeAdapter` has `stdin: ReadFn` and `stdinIsTTY: boolean`. `ReadFn` is `() => Promise<string | null>` (from prompt module). `createNodeAdapter` wraps `process.stdin` via dynamic `import('node:readline')` — lazy per-call readline interfaces avoid holding stdin open. `createTestAdapter` defaults to EOF-returning stdin (`() => Promise.resolve(null)`) and `stdinIsTTY: false`. `CLIBuilder.run()` auto-creates a `createTerminalPrompter(adapter.stdin, adapter.stderr)` when `stdinIsTTY=true` and no explicit prompter provided — this is the prompt gating seam. Prompts write to stderr so they don't interfere with piped stdout. `node-builtins.d.ts` provides minimal `node:readline` types to avoid `@types/node`.
- **Interactive resolver pattern**: `CommandBuilder.interactive(resolver)` stores a type-erased `ErasedInteractiveResolver` on `CommandSchema.interactive`. Resolver receives `{ flags: Partial<InferFlags<F>> }` (partially resolved after CLI/env/config). Returns `Record<string, PromptConfig | false | undefined | null | 0 | ''>`. Semantics: truthy `PromptConfig` overrides per-flag prompt; `false` explicitly suppresses prompting; `undefined`/`null`/absent falls back to per-flag `.prompt()` config. Resolution is two-pass: pass 1 resolves CLI/env/config for all flags; interactive resolver called with partial state; pass 2 prompts unresolved flags using interactive configs or per-flag fallback, then applies defaults/required. Type erasure via `as unknown as ErasedInteractiveResolver` — single cast point, same pattern as handler erasure in testkit.
- **WidenContext conditional type**: `Record<string, never>` as default C makes naive intersection collapse to `never`. Use `WidenContext<C, Output> = C extends Record<string, never> ? Output : C & Output` to replace on first middleware, intersect on subsequent.
- **Middleware explicit generic**: `middleware<{ user: string }>(...)` is needed for reliable inference in strict TS mode; inference from `next()` argument alone is unreliable across all TS contexts.
- **Middleware phantom type via cast**: `MiddlewareImpl` (runtime) cast to `Middleware<Output>` (phantom-branded) — same pattern as flag/arg builders.
- **executeWithMiddleware continuation chain**: Build from back to front so first middleware is outermost. Each `next()` closes over downstream chain.
- **Global flag pattern**: Detect in `CLIBuilder.execute()`, strip from argv before dispatch, propagate via options to downstream `runCommand()`. No global flag schema — hardcoded like `--version`/`--help`.
- **JSON mode redirection**: In JSON mode, `log`/`info` → stderr, `json()` → stdout. Errors rendered via `out.json({ error: err.toJSON() })` to stdout. Scripts get one structured JSON stream on stdout.
- **Table output dual-mode**: `table()` in JSON mode delegates to `json(rows)` (full data, unfiltered). In normal mode, delegates to `log(formatTable(...))` so redirection rules apply automatically.
- **isTTY wiring pattern**: `adapter.isTTY` → `CLIRunOptions.isTTY` → `RunOptions.isTTY` → `createCaptureOutput({ isTTY })` → `OutputChannel.isTTY` → `Out.isTTY`. Each layer conditionally spreads to respect `exactOptionalPropertyTypes`.
- **Output mode precedence**: `jsonMode > isTTY` — handlers check `jsonMode` first, then `isTTY`.
- **CLIBuilder.command() must accept any C**: `eraseCommand()` and `CLIBuilder.command()` need `C extends Record<string, unknown>` generic param so commands with middleware can be registered.
- **JSON error rendering at every error site**: Both `CLIBuilder.execute()` (dispatch errors) and `runCommand()` (command errors) have separate error paths. Each must check `jsonMode`.

The default type parameter on the class propagates to all method return types, so changing `CommandBuilder<F, A>` → `CommandBuilder<F, A, C>` with `C = Record<string, never>` is fully backward-compatible — existing code that doesn't specify `C` gets the default. `expectTypeOf<T>().toBeNever()` doesn't work when the value IS `never` — use `type CtxValue = (typeof ctx)[string]; expectTypeOf<CtxValue>().toBeNever()` instead.
