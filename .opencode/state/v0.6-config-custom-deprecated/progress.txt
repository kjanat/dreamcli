## Codebase Patterns

- **RuntimeAdapter extension pattern**: Add field to interface ‚Üí add to TestAdapterOptions with sensible default ‚Üí implement in createTestAdapter ‚Üí implement in createNodeAdapter ‚Üí add `platform` to NodeProcess if needed ‚Üí update all mock NodeProcess objects in tests ‚Üí add test section for new fields
- **Homedir/configDir from env vars**: Avoid `node:os` import; derive from `process.env` (HOME, USERPROFILE, XDG_CONFIG_HOME, APPDATA) + `process.platform` to keep factory synchronous
- **readFile ENOENT ‚Üí null**: Use `isNodeSystemError` type guard to detect `{ code: 'ENOENT' }` without `@types/node`; rethrow all other errors
- **node-builtins.d.ts**: Add minimal type shims for any new `node:*` imports (only the subset actually used)
- **Config discovery pattern**: Pure function taking `Pick<RuntimeAdapter, 'readFile' | 'cwd' | 'configDir'>` ‚Äî narrow adapter pick for testability. Discriminated union result (`found: true/false`), errors thrown as `CLIError` with specific codes. Extension-based loader dispatch with JSON built-in.
- **Path separator inference**: Detect `\` vs `/` from adapter-provided paths (already platform-native) to avoid `node:path` import. `joinPath(base, ...segments)` helper.
- **Virtual filesystem testing**: `stubAdapter({ '/path': 'content' })` pattern for config tests ‚Äî maps paths to file content, returns `null` for missing.
- **Global flag extraction in `.run()`**: Extract flags that need I/O (like `--config <path>`) in `.run()` before calling `.execute()`. Follows same pattern as `env`/`isTTY` auto-sourcing ‚Äî `.run()` does adapter-dependent work, passes result to pure `.execute()`.
- **CLISchema extension pattern**: Add field with `| undefined` default ‚Üí update `cli()` factory to include it ‚Üí add builder method ‚Üí conditionally act on it in `.run()`/`.execute()`.
- **Config errors before command dispatch**: Detect `--json` from raw argv before config loading to render errors in correct format (JSON vs text).
- **Plugin hook pattern**: Incremental builder method `.configLoader(loader)` appends to `configSettings.loaders`. Requires `.config()` first (throws `INVALID_BUILDER_STATE` otherwise). `configFormat(extensions, parseFn)` convenience factory avoids manual object construction.
