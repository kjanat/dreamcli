## Codebase Patterns
<!-- Consolidate reusable patterns here -->
- `Out` interface in `src/core/schema/command.ts` — all output methods defined here
- `OutputChannel` in `src/core/output/index.ts` — implements `Out`, routes by jsonMode/isTTY/verbosity
- Factory: `createOutput()` for production, `createCaptureOutput()` for testkit
- `CapturedOutput` has `stdout: string[]`, `stderr: string[]`, `activity: ActivityEvent[]` — push-based capture
- `RunResult` in `src/core/testkit/index.ts` — exitCode, stdout, stderr, error
- Barrel re-exports: module `index.ts` → `schema/index.ts` → `src/index.ts`
- Types use `readonly` everywhere, interfaces over type aliases for shapes
- `@internal` JSDoc excludes from public API
- Co-located test files: `*.test.ts` next to source
- `.js` extensions on all relative imports (NodeNext)
- `import type` for type-only imports (verbatimModuleSyntax)

- When extending `Out`, all mock objects in tests must be updated (command.test.ts had `mockOut: Out`)
- Noop handle singletons in `output/index.ts` for silent mode; static handle classes for plain-text fallback
- Static handles take `StaticWriters` (stdout/stderr `WriteFn` pair), not `Out`, to avoid circular dep
- TTY handles also use `StaticWriters` — same pattern, consistency across all concrete handle types
- Timer APIs (`setInterval`/`clearInterval`) need `declare function` stubs since lib is ES2022-only (no DOM/Node typings)
- ANSI constants are module-level `const` with `@internal` JSDoc — no namespace, flat declarations
