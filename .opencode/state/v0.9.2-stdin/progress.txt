# Progress Log

PRD: v0.9.2-stdin Started: 2026-03-30

## Codebase Patterns

- `RuntimeAdapter` interface in `src/runtime/adapter.ts` — all platform-dependent ops
- `createTestAdapter()` in same file — test stub, all fields optional with sensible defaults
- Node adapter: `src/runtime/node.ts`, Bun: `src/runtime/bun.ts` (delegates to Node), Deno: `src/runtime/deno.ts`
- `ReadFn` from `core/prompt/` = line reader (single-line, for prompts), NOT bulk stdin
- Stdin wiring pattern: `RunOptions`/`CLIRunOptions` carry `stdinData`, `CLIBuilder.run()` reads adapter stdin once, and `resolve()` threads it into `resolveArgs()`
- Resolver-level precedence tests are simplest when they build a real command schema, call `parse(schema, argv)`, then assert on `resolve()` directly
- `ArgSchema` in `src/core/schema/arg.ts` — runtime descriptor for args
- Build-time arg invariants live in `CommandBuilder.arg()` via schema inspection + `CLIError` codes, not in `ArgBuilder`
- `resolveArgs()` in `src/core/resolve/index.ts` — CLI → stdin → env → default chain for args
- `ResolveOptions` in `src/core/resolve/index.ts:42` — injectable state for resolver
- `RunOptions` in `src/core/testkit/index.ts:66` — testing seam
- `CLIRunOptions` in `src/core/cli/index.ts:174` — runtime seam for `cli().run()`
- Convention: `exactOptionalPropertyTypes` forces conditional spread `...(x !== undefined ? { x } : {})`
- Convention: `.ts` extensions in all relative imports
- Convention: `import type` for type-only imports (verbatimModuleSyntax)

## Task - adapter-readStdin

- Added `readStdin(): Promise<string | null>` to `RuntimeAdapter` interface
- Node: `readNodeStdinAll()` — async iterator over `process.stdin` chunks, TextDecoder streaming
- Deno: `readDenoStdinAll()` — `ReadableStream.getReader()` loop, TextDecoder streaming
- Bun: delegates to Node (unchanged pattern)
- Test adapter: `stdinData?: string` on `TestAdapterOptions`, wired in `createTestAdapter()`
- Both Node and Deno return `null` immediately when `stdinIsTTY` is true
- Added `[Symbol.asyncIterator]` to `NodeProcess.stdin` type — required for `for await`
- Fixed test helpers in `runtime.test.ts` and `bun.test.ts` — `mockStdin()` factory with async iterator
- **Files changed:** `adapter.ts`, `node.ts`, `deno.ts`, `runtime.test.ts`, `bun.test.ts`
- **Learnings:** `exactOptionalPropertyTypes` + `Partial<T>` is painful for overrides with required nested properties — use `Omit<Partial<T>, 'key'> & { key?: T['key'] }` pattern to preserve exact type for the nested property

## Task - arg-stdin-modifier

- Added `stdinMode` to `ArgSchema` with default `false` and added `ArgBuilder.stdin()` as a no-type-change modifier
- Enforced build-time arg invariants in `CommandBuilder.arg()`:
  duplicate stdin args throw `CLIError` code `DUPLICATE_STDIN_ARG`
  `stdin + variadic` throws `CLIError` code `INVALID_BUILDER_STATE`
- Added focused schema tests for `.stdin()` and command builder validation cases
- **Files changed:** `src/core/schema/arg.ts`, `src/core/schema/arg.test.ts`, `src/core/schema/command.ts`, `src/core/schema/command.test.ts`, `.opencode/state/v0.9.2-stdin/prd.json`, `.opencode/state/v0.9.2-stdin/progress.txt`
- **Learnings:** keep arg-shape validation at command registration time so invalid cross-arg combinations are rejected once, before parse/resolve logic grows around them

## Task - resolve-stdin-wire

- Added `stdinData?: string | null` to `ResolveOptions`, `RunOptions`, and `CLIRunOptions`

- Updated `CLIBuilder.run()` to call `adapter.readStdin()` once and pass the result into execute-time options
- Added focused integration coverage for `runCommand()`, `CLIBuilder.execute()`, and `CLIBuilder.run()` stdin threading
- **Files changed:** `src/core/resolve/index.ts`, `src/core/testkit/index.ts`, `src/core/cli/index.ts`, `src/core/resolve/resolve-integration.test.ts`, `.opencode/state/v0.9.2-stdin/prd.json`, `.opencode/state/v0.9.2-stdin/progress.txt`
- **Learnings:** for stdin-backed args, treat explicit `'-'` as a source selector rather than a literal CLI value so the normal fallback chain can continue when no piped data exists
